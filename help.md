# Asset Curator – User Guide

## Introduction

Asset Curator is an asset management application designed to track, manage, and analyse physical assets using QR codes, structured data, maintenance tracking, and reporting.

---

## Getting Started

When you first open the app, it operates in **Local Mode** by default.

### Free Version

* You can immediately begin adding and managing assets using a **local database stored on your device**
* Sync features are **not available** in the free tier

### Paid Versions (Premium and above)

Once upgraded, additional options become available:

* Connect to an existing **Master Database**
* Create a **Shared Master Database** for multi-device use
* Sync between local and master databases

This allows multiple users and devices to work with the same asset dataset.

## Main Screen

The main screen displays all assets with:

* Title and Asset ID
* Current Location and Home Location
* Status
* Optional Image

### Actions

* Scan Label
* Add Item
* Print Label

### Tools

* Search (Title, ID, Tags)
* Filters (Tag, Location)
* View modes (List / No Images)

---

## Adding an Asset

### Required Fields

* Asset ID (auto-generated)
* Title
* Home Location
* Current Location
* Status
* Tags

### Optional Fields

* Description
* Image
* Serial Number
* Part Number
* Purchase / Sale details
* Record last updated

---

## Locations & Tags

### Locations

Used to track where items belong and where they currently are.

### Tags

Used for categorisation and filtering.

You can:

* Add
* Rename
* Deactivate
* Delete (if not in use)

---

## QR Codes & Labels

Each asset includes a QR code.

* Print labels (Avery / Dymo supported)
* Scan to open the asset instantly
* Can redirect to app install if not installed

---

## Search & Filtering

* Search by Title, Asset ID, or Tags
* Filter by Location or Tag
* Switch between image and no-image views

---

## Sync (Database Modes)

### Local Mode

* Data stored only on device

### Shared Mode

* Uses a Master Database file
* Supports multiple users/devices
* Sync uses timestamps to resolve changes

### Setup Steps

1. Choose Master Folder
2. Create Master Database
3. Create Local Database
4. Sync Databases
